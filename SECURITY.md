# Security Policy

## Supported versions

Security updates are applied to the latest `master` branch state.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities privately with:

- A clear description of the issue
- Reproduction steps
- Impact assessment
- Suggested mitigation (if known)

After confirmation, maintainers will:

1. Acknowledge receipt.
2. Triage severity.
3. Prepare a fix.
4. Coordinate disclosure.

## Disclosure policy

We follow responsible disclosure and aim to release patches before publishing full technical details.

## Dependency hygiene

- Keep dependencies up to date.
- Avoid introducing unmaintained libraries.
- Review network/auth changes carefully.
